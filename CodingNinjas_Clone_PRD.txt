1. Project Overview

This document defines the full product requirements for building a pixel-perfect, fully functional clone of the Coding Ninjas website (codingninjas.com) using React.js and Tailwind CSS. The goal is to replicate the site's visual design, information architecture, and key interactive behaviors at a production-ready standard.

1.1 Project Goals
Faithfully replicate the UI/UX of codingninjas.com homepage and core pages
Build a scalable, component-driven architecture using React + Tailwind CSS
Implement responsive design for mobile, tablet, and desktop viewports
Deliver clean, maintainable code that mirrors modern industry standards
Achieve lighthouse performance score of 85+ across all pages

1.2 Scope
In Scope:
Homepage (full section-by-section clone)
Course listing page with filter tabs
Individual course detail page
Authentication pages (Login / Signup)
About Us page
Responsive Navigation (Navbar + Mobile drawer)
Footer with all link groups
Out of Scope:
Backend / API development
Payment gateway integration
Real user authentication
Admin dashboard
Actual video content playback

4.2 Homepage Sections
The homepage is the primary landing page and consists of 10+ distinct sections. Each must be built as a standalone React component.

4.2.1  Hero Section
Headline: "Give your career an unfair AI advantage"
Subheading with AI tools icons marquee (infinite horizontal scroll strip)
AI tool logos: Google Gemini, LangChain, FastAPI, ChatGPT, CrewAI, Node.js, LLM
Course Finder Form: Experience dropdown, Topic of Interest selector, Name/Phone/Email fields, Submit CTA
Animated recommendation card shown on form submit
Background: Dark gradient with orange accent elements

4.2.2  Explore Courses Section
Tab navigation: Popular / Data Analytics / Generative AI / Software Development / Data Science
Secondary tab: For Working Professionals / For College Students
Course cards grid (3 columns desktop, 2 tablet, 1 mobile)
Each card: Banner image, category badge, partner logo, course title, duration, enrollment count, AI tools count, type badge
Hover effect: Slight elevation / shadow
CTA: "Visit course page" button linking to /courses/:slug

4.2.3  Build with AI Section
Heading: "Build with AI like never before"
Auto-scrolling horizontal carousel of project cards (US Healthcare, Credit Risk, Meal Plan, etc.)

11. Accessibility Requirements

All interactive elements must be keyboard navigable (Tab, Enter, Escape)
Semantic HTML: <nav>, <main>, <section>, <article>, <header>, <footer>
All images must have descriptive alt text
Color contrast ratio >= 4.5:1 for all text on backgrounds
aria-label on icon-only buttons
aria-expanded on accordion and dropdown triggers
Focus-visible rings on all interactive elements (outline-2 offset-2)
Skip-to-main-content link at top of page
Modal traps focus correctly and restores on close
Carousel supports keyboard arrow navigation

14. Acceptance Criteria

The following criteria must be met before the project is considered complete:

All pages listed in the sitemap (Section 3) are implemented and navigable
Homepage renders all 9 sections with visual parity to codingninjas.com
Responsive layout works correctly at 375px, 768px, 1024px, and 1440px viewports
All interactive elements (dropdowns, tabs, carousels, accordions, modals) are functional
Course filter tabs correctly show/hide course cards by category
Navigation dropdown menus work on desktop hover and mobile tap
Login and signup forms validate all fields with appropriate error messages
Infinite marquee animations run smoothly without jank (60fps)
All stat counters animate when scrolled into view
Lighthouse performance score >= 85 on production build
No console errors or warnings in production build
Site passes WCAG 2.1 AA keyboard navigation and contrast requirements
All images are lazy-loaded and in WebP format
Code passes ESLint with zero errors and is Prettier formatted


15. Risks & Mitigations

Risk
Mitigation
Image assets unavailable
Use placeholder images from Unsplash / Picsum; document real URLs
Font licensing
Use Google Fonts Inter (open license)
Animation performance on mobile
Reduce motion via prefers-reduced-motion media query
Carousel library conflicts
Test Swiper.js early; fallback to plain CSS scroll snap
Complex responsive layout drift
Use Tailwind responsive prefixes consistently; test at every breakpoint
Mock data mismatch
Structure JSON to exactly mirror real API response shapes
Bundle bloat from icon libraries
Import icons individually, never entire library
